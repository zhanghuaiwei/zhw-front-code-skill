## 基础格式化

| 规则项 | 规范值 | 严格程度 |
|--------|--------|---------|
| 缩进 | 2 个空格，**禁止使用 tab** | 🔴 强制 |
| 分号 | 不需要分号 | 🟡 推荐 |
| 空格 | 运算符前后一致使用单空格 | 🔴 强制 |
| 引号 | JS/TS 使用单引号，HTML/CSS 使用双引号 | 🔴 强制 |
| 行宽 | 单行不超过 120 字符 | 🟡 推荐 |
| 逗号 | 后面必须加空格，前面不加 | 🔴 强制 |
| 空行 | 逻辑块之间加空行，提高可读性 | 🟡 推荐 |

> 💡 分号规则遵循 **Vue 3 官方编码风格**

## TypeScript / JavaScript 规范

✅ **强制校验工具**：
- ESLint 完整校验
- TypeScript 严格模式 + 编译检查

### 核心规则

1. **🔴 编译层面禁止 `any` 类型**
2. **🔴 所有函数必须声明返回类型**，禁止依赖自动推导
3. 🟡 优先使用 `const`，其次 `let`，禁止使用 `var`
4. 🟡 使用箭头函数代替普通函数
5. 🟡 使用解构赋值提取对象属性
6. 🟡 使用模板字符串代替字符串拼接

> 不接受反驳：自行推导不符合类型规范安全，我要尽可能像 Java 一样，每个变量、方法都能找到对应的类型。

## CSS / 样式规范

✅ **强制校验工具**：StyleLint

| 规则项 | 规范值 |
|--------|--------|
| 选择器命名 | BEM 命名法 |
| 样式单位 | 优先使用 rem / px |
| 属性顺序 | 布局类 → 盒模型 → 文字类 → 装饰类 |
| 颜色值 | 使用小写，优先简写 |
| 嵌套层级 | 不超过 3 层 |

## Vue 风格规范

严格遵循 **Vue 3 官方风格指南**：

1. 组件名为多个单词
2. Prop 定义尽量详细
3. 为 `v-for` 设置键值
4. 模板中使用简单表达式，复杂逻辑抽离到 computed
5. SFC 单文件组件统一结构：`<script setup>` → `<template>` → `<style>`

## React 风格规范

严格遵循 **React 官方风格指南**：

1. 组件名使用大驼峰，实例使用小驼峰
2. 始终明确声明 PropTypes / TypeScript 类型
3. JSX 中使用双引号
4. 对齐方式遵循官方示例
5. 自定义 Hook 以 `use` 开头
