## 整体结构总览

```
web/
├── dist/                          # 构建输出目录
├── src/                           # 源代码目录
│   ├── api/                       # API 接口层
│   ├── assets/                    # 静态资源
│   ├── components/                # 🔒 无业务纯UI公共组件
│   ├── constants/                 # 常量定义
│   ├── enums/                     # 枚举类型
│   ├── hooks/                     # 全局自定义 Hooks
│   ├── layouts/                   # 布局组件
│   ├── stores/                    # 全局状态
│   ├── styles/                    # 全局样式
│   ├── utils/                     # 工具函数
│   ├── views/                     # 📄 页面组件
│   │   ├── components/            # 📦 可复用的业务组件
│   │   └── user/                  # 单个页面模块（页面自治）
│   ├── App.tsx / App.vue          # 应用入口组件
│   └── main.ts                    # 应用启动文件
├── types/                         # 全局 TypeScript 类型
├── tests/                         # 测试用例
├── .env                           # 环境变量（通用）
├── .env.development               # 开发环境变量
├── .env.development.local         # 本地开发环境变量（不提交）
├── .env.production                # 生产环境变量
├── .env.test                      # 测试环境变量
├── .eslintrc.cjs                  # ESLint 配置
├── .stylelintrc.cjs               # StyleLint 配置
├── Dockerfile                     # Docker 构建配置
├── nginx.conf                     # Nginx 配置
├── package.json
├── tsconfig.json
└── vite.config.ts
```

## 详细目录说明

### 1. `/src/api/` - API 接口层

```
api/
├── modules/
│   ├── auth.ts                    # 认证模块接口
│   └── user.ts                    # 用户模块接口
└── request.ts                     # Axios 实例 + 全局拦截器
```

**原则**：按业务模块化组织，统一错误处理，统一数据转换。

---

### 2. `/src/components/` - 纯UI公共组件

```
components/
├── ActionButtons/                 # 操作按钮组
├── PageHeader/                    # 页面头部
├── StatusTag/                     # 状态标签
└── Table/                         # 增强表格
```

**🔴 红线原则**：这里的组件**绝对不能包含业务逻辑**，只能接收 props 渲染。

---

### 3. `/src/views/` - 页面组件区

```
views/
├── components/                    # 📦 跨页面复用的业务组件
│   └── UserSelect/                # 用户选择器（业务相关）
└── user/
    ├── components/                # 只给 user 页面用的组件
    ├── composables/               # 当前页面自治的逻辑
    │   └── useList.ts             # 列表业务逻辑
    ├── api.ts                     # 当前页面私有 API
    ├── types.ts                   # 当前页面私有类型
    └── index.vue                  # 页面入口
```

**页面自治原则**：一个页面自己用的东西，就放在自己目录下，不要提到全局。

## 组件分层原则

```
┌─────────────────────────────────────────────────────────┐
│  views/user/components/                                 │
│  只给 user 页面用的业务组件                              │
├─────────────────────────────────────────────────────────┤
│  views/components/                                      │
│  多个页面共用的业务组件                                  │
├─────────────────────────────────────────────────────────┤
│  src/components/                                        │
│  无业务的纯UI组件（全站通用）                            │
└─────────────────────────────────────────────────────────┘
```

> 💡 这条线吃过亏，破过防——所以现在一定要做成范式，坚决守住。

- ✅ 下层的组件可以被上层调用
- ❌ 上层的组件绝对不能被下层调用
- ❌ 禁止反向依赖

## 页面自治原则

每个页面模块是独立自治的：

```
user/
├── components/        # 组件只给我用
├── composables/       # 逻辑只给我用
├── api.ts             # API 只给我调
├── types.ts           # 类型只给我用
└── index.vue          # 我是入口
```

**好处**：
1. 删除一个功能时，直接删除整个文件夹，干净不留痕
2. 新人维护时，只需要看这个文件夹
3. 不会出现"这个文件还有人用吗"的问题
